# Инструкция по настройке контроллера ворот
# Gate Controller v4.2 — Руководство по параметрам

---

## Быстрый старт после прошивки

1. Выставить `PULSES_OPEN = 0` и `PULSES_CLOSE = 0`
2. Прошить — контроллер войдёт в режим калибровки
3. Открыть ворота кнопкой (один раз полный ход)
4. Закрыть ворота кнопкой (один раз полный ход)
5. В Serial Monitor появится:
   ```
   CALIBRATION DONE: OPEN=XX CLOSE=XX
   ```
6. Вписать эти числа в код и перепрошить

## Параметры мощности

### Если ворота едут быстрее или медленнее 15 секунд
Менять только крейсерскую мощность:
```cpp
const int OPEN_POWER_CRUISE  = 38;  // открытие
const int CLOSE_POWER_CRUISE = 40;  // закрытие
```
- Ворота едут **быстрее 15с** → уменьшить на 3–5%
- Ворота едут **медленнее 15с** → увеличить на 3–5%
- Из логов: при 38% открытие ~17с, при 43% будет ~15с

### Если ворота не стартуют с места (особенно закрытие)
```cpp
const int CLOSE_POWER_KICK = 42;    // мощность первых 800мс закрытия
const int OPEN_POWER_KICK  = 40;    // мощность первых 300мс открытия
```
Увеличить на 5–10% если мотор не трогается.

### Если ворота не закрывают замок до конца (не хватает усилия)
```cpp
const int CLOSE_POWER_SLOW = 48;    // мощность в последних 22% хода закрытия
```
Увеличить до 55–60%. Этот параметр отвечает за усилие при замыкании замка
и продавливание утеплителя. При закрытии замедление намеренно выше крейсера.

### Если сильно бьёт в концевик при открытии
```cpp
const int OPEN_POWER_SLOW = 35;     // мощность в последних 22% хода открытия
```
Уменьшить до 25–28%.

## Тайминги профиля

```cpp
const unsigned long KICK_MS       = 300;   // длительность стартового пинка открытия (мс)
const unsigned long CLOSE_KICK_MS = 800;   // длительность стартового пинка закрытия (мс)
const unsigned long RAMP_UP_MS    = 2000;  // время плавного разгона после пинка (мс)
const float         SLOWDOWN_AT   = 0.22;  // замедление на последних 22% пути (доля 0..1)
```

- Старт **слишком резкий** → увеличить `KICK_MS` / `CLOSE_KICK_MS`
- Ворота **долго разгоняются** → уменьшить `RAMP_UP_MS`
- Замедление **начинается слишком рано** → уменьшить `SLOWDOWN_AT` (например 0.15)
- Замедление **слишком позднее** → увеличить `SLOWDOWN_AT` (например 0.30)

## Коррекция скорости (адаптация к наледи и нагрузке)

```cpp
const int MAX_CORRECTION      = 8;    // максимальное отклонение мощности от профиля (%)
const int POWER_STEP          = 2;    // шаг коррекции за каждые 600мс (%)
const float CORRECTION_DEADBAND = 0.06; // погрешность 6% — не корректировать
```

Коррекция работает только в крейсерской фазе (не во время разгона и замедления).
- `MAX_CORRECTION = 8` → мощность может отклоняться от профиля не более чем на ±8%
- Увеличить если ворота сильно тормозят от наледи и коррекции не хватает
- Уменьшить если ворота "дёргаются" при коррекции

## Защита от застревания

```cpp
const unsigned long STALL_TIMEOUT = 2500;  // нет импульсов X мс → стоп (мс)
```

Проверка начинается через 1 секунду после пинка — чтобы не срабатывать при старте.
- Увеличить если ворота иногда останавливаются без причины (ложное срабатывание)
- Уменьшить если нужно быстрее реагировать на препятствие

## Датчик оборотов (геркон)

```cpp
const unsigned long DEBOUNCE_TIME_US = 8000;  // антидребезг 8мс
```

**Критически важный параметр!** При значении 500 (заводское) датчик давал
сотни ложных импульсов — ПИД сходил с ума. При 8000 импульсы чистые.
- Не уменьшать ниже 5000
- Если импульсы пропадают при высокой скорости — уменьшить до 6000

## Импульсы (калибровка)

```cpp
unsigned long PULSES_OPEN  = 114;  // импульсов от закрытого до открытого концевика
unsigned long PULSES_CLOSE = 120;  // импульсов от открытого до закрытого концевика
```

Эти значения определяют где начинается замедление и как работает коррекция.
Получаются автоматически при калибровке (PULSES = 0).
**После замены или перемещения датчика — повторить калибровку.**

## Защёлка (костыль для открытия)

```cpp
const unsigned long LATCH_DELAY        = 1000;  // время удержания реле защёлки (мс)
const unsigned long PUSH_DELAY         = 120;   // время пинка мотором (мс)
const unsigned long LATCH_WAIT_TIMEOUT = 4000;  // ожидание отпускания концевика (мс)
```

При неудачной попытке открытия контроллер автоматически повторяет один раз.
Если две попытки не удались — остановка (событие `LATCH_FAILED` в Serial).

- Замок не открывается → увеличить `LATCH_DELAY` до 1500
- Концевик не отпускает → увеличить `LATCH_WAIT_TIMEOUT` до 5000

## Диагностика через Serial Monitor (115200 бод)

Формат CSV лога между `---CSV_START---` и `---CSV_END---`:
```
time_ms, ses_ms, dir, power, offset, pulses, exp%, act%, event
```
- `dir` — O=открытие, C=закрытие
- `power` — текущая мощность %
- `offset` — поправка коррекции (+ = добавили газ, - = убрали)
- `exp%` — где должны быть по времени
- `act%` — где реально находимся по импульсам

**Признаки проблем:**
- `offset` уходит в -8 и держится → ворота едут быстрее нормы, снизить `CRUISE`
- `offset` уходит в +8 и держится → ворота едут медленнее нормы, поднять `CRUISE`
- `STALL` без препятствий → мощность слишком низкая в конце хода, поднять `POWER_SLOW`
- Много строк с одинаковым `pulses` → датчик не читает, проверить зазор магнита
